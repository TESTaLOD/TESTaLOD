This section explains how to structure test case owl files to make them compatible with the TESTaLOD methodology and application.

The property "hasSPARQLQueryUnitTest" (taken form testannotationschema ontology) refers to an escaped SPARQL query that describes a competency question to test an ontology:

                                
<testannotationschema:hasSPARQLQueryUnitTest xml:lang="en">
    PREFIX arco-dd: &lt;https://w3id.org/arco/ontology/denotative-description/&gt;

    SELECT *
    WHERE{
        ?cultprop arco-dd:hasAffixedElement ?AffElem.
        ?AffElem a arco-dd:CoinLegend . 
    }
    LIMIT 100
</testannotationschema:hasSPARQLQueryUnitTest>

To perform an ontology test it is necessary to have a list of results with which the competency question results will be compared to determine the test results.

The property "hasExpectedResult" (taken form testannotationschema ontology) describes a list of desired SPARQL results in an escaped JSON format (as described by W3C Recommendation)

                    
<testannotationschema:hasExpectedResult xml:lang="it">
    {
        &quot;head&quot;: {
            &quot;vars&quot;: [ &quot;cultprop&quot; , &quot;AffElem&quot; ]
        } ,
        &quot;results&quot;: {
            &quot;bindings&quot;: [
                {
                    &quot;cultprop&quot;: { &quot;type&quot;: &quot;uri&quot; , &quot;value&quot;: &quot;https://w3id.org/arco/test/VI/ToyDatasets/Toyset_VI06.owl#MonetaBaiocco&quot; } ,
                    &quot;AffElem&quot;: { &quot;type&quot;: &quot;uri&quot; , &quot;value&quot;: &quot;https://w3id.org/arco/test/VI/ToyDatasets/Toyset_VI06.owl#LegendaMonetaBaiocco&quot; }
                }
            ]
        }
    }
</testannotationschema:hasExpectedResult>

Finally, SPARQL query requires some data to run the test. It must be described how and where to perform the test.

This is done through TESTaLOD experimental ontology which provides two possibilities:

1. SPARQL Endpoint

If data are available through a SPARQL endpoint, the TESTaLOD application can run the test directly on this endpoint. To do this it must be described the type of input data through the "hashasInputTestDataCategory" property (TESTaLOD ontology provides an input data categories vocabulary). In this case the given input category will be: "#SPARQLendpoint". Also the SPARQL endpoint URI must be provided, through the "hasInputTestDataUri" property. 

                    
<testalod:hasInputTestDataCategory rdf:resource="https://raw.githubusercontent.com/TESTaLOD/TESTaLOD/master/ontology/testalod.owl#SPARQLendpoint"/>
<testalod:hasInputTestDataUri rdf:resource="http://wit.istc.cnr.it/arco/virtuoso/sparql"/>
2. ToyDataset

Otherwise, if data are not yet available on a SPARQL endpoint, it is possible to perform the test on a toy dataset, an OWL file in which a dataset is simulated to test the ontology. In this case the "hasInputTestDataCategory" property will refer to "#ToyDataset", while the "hasInputTestDataUri" property will indicate toy dataset URI (i.e. Toy Dataset can be uploaded on Github) 


                    
<testalod:hasInputTestDataCategory rdf:resource="https://raw.githubusercontent.com/TESTaLOD/TESTaLOD/master/ontology/testalod.owl#ToyDataset"/>
<testalod:hasInputTestDataUri rdf:resource="https://raw.githubusercontent.com/ICCD-MiBACT/ArCo/master/ArCo-release/test/VI/ToyDatasets/Toyset_VI06.owl"/>
